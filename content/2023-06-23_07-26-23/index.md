## 目的

可読性をあげること

## 経緯

商品の値段を表すprice変数の値の処理で
値がマイナスだったら0をセットするという処理
これが読みづらいなとずっと思ってて

## 3項演算子でこうやって書いてたけど

```js
price = price < 0 ? 0 : price;
```

## こっちの方が見やすい！

```js
price = Math.max(price, 0);
```

## 学び

今までリファクタリング時に変数名をばかり注視してた
処理方法については、でかい処理はカスタム関数作るが
小さい処理はあえて関数作っても逆に読みにくいと思ってた
小さい処理は、標準関数が使えないか考えるべきだった
